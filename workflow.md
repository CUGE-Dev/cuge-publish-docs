# 用户文档写作流程

## 1. 安装本地文档预览依赖

终端输入

```shell
npm i docsify-cli -g
```

确认

```shell
docsify --help
```

能正常输出帮助信息。

## 2. 克隆本仓库

这个不多说。

## 3. 在对应的目录下写作

| 平台                     | 对应文件夹   |
| ------------------------ | ------------ |
| 服务端                   | `server`     |
| 客户端                   | `client`     |
| 管理后台                 | `management` |
| 杂项(比如说这个规范文档) | 根目录       |

## 4. 添加侧边栏

编辑 `_sidebar.md`文件，在对应分栏下添加文档引用，具体格式可以参考现存的`_sidebar.md`文件。

## 5. 本地验证

在文档根目录下执行

```shell
docsify serve .
```

本地预览看看文档是否正常。

## 6. 发布

推送工作分支，稍等片刻，刷新地址，预览发布到托管平台的文档是否正常。
